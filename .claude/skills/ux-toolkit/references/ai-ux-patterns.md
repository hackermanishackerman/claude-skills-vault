# AI/ML UX Patterns

Evaluate AI-powered interfaces for transparency, trust, control, and error handling.

## Transparency & Explainability

### AI Disclosure Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| AI presence disclosed | Users know AI is involved | Critical |
| Capability boundaries | Clear what AI can/cannot do | Major |
| Data usage explained | How user data trains/improves AI | Major |
| Source attribution | Where AI information comes from | Major |
| Confidence indication | Certainty levels shown when relevant | Minor |

### Disclosure Patterns

| Pattern | When to Use | Example |
|---------|-------------|---------|
| Inline label | AI-generated content | "Generated by AI" badge |
| Onboarding | First-time AI feature use | "This feature uses AI to..." |
| Tooltip | On hover/focus | "AI-suggested based on..." |
| Settings | Configuration page | "AI features: On/Off" |

### Explainability Levels

| Level | Description | Example |
|-------|-------------|---------|
| **Basic** | "AI suggested this" | Recommendation badge |
| **Intermediate** | "Based on your history" | Key factors shown |
| **Advanced** | "Here's why + alternatives" | Full reasoning + options |

### Explainability Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Why explanation | Reason for AI decision available | Major |
| Key factors | Most important inputs highlighted | Minor |
| Counterfactuals | "What would change result" | Cosmetic |
| Layered depth | Simple ‚Üí detailed drill-down | Minor |
| Non-technical | Plain language explanations | Major |

## Trust & Confidence

### Trust Indicators

| Indicator | Implementation | Purpose |
|-----------|----------------|---------|
| Confidence score | "85% confident" | Set expectations |
| Source citation | "Based on 3 sources" | Verifiability |
| Recency indicator | "Updated 2 hours ago" | Freshness |
| Human review badge | "Verified by expert" | Quality assurance |
| Uncertainty admission | "I'm not sure about..." | Honest limitations |

### Trust Building Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Consistent accuracy | AI delivers on promises | Critical |
| Graceful uncertainty | Admits when unsure | Major |
| No overconfidence | Doesn't claim certainty falsely | Critical |
| Recoverable errors | Easy to correct AI mistakes | Major |
| Learning feedback | User can teach AI | Minor |

### Confidence Display Patterns

```
High Confidence (>90%):
  "This is a cat" ‚úì

Medium Confidence (60-90%):
  "This looks like a cat (85% confident)"

Low Confidence (<60%):
  "This might be a cat, but I'm not sure.
   Other possibilities: dog, fox"

No Confidence:
  "I can't determine what this is.
   Try uploading a clearer image."
```

## User Control & Agency

### Control Requirements

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Opt-out available | Can disable AI features | Critical |
| Manual override | Can reject AI suggestions | Critical |
| Edit AI output | Can modify generated content | Major |
| Undo/revert | Can return to pre-AI state | Critical |
| Preference memory | AI respects user preferences | Minor |

### Control Patterns

| Pattern | Description | Example |
|---------|-------------|---------|
| Accept/Reject | Binary choice on suggestions | "Use suggestion" / "Ignore" |
| Edit in place | Modify AI output directly | Editable generated text |
| Regenerate | Request new AI output | "Try again" button |
| Feedback loop | Train AI from corrections | "This was wrong because..." |
| Kill switch | Disable AI entirely | Settings toggle |

### Autonomy Levels

| Level | AI Role | Human Role | Example |
|-------|---------|------------|---------|
| Recommendation | Suggests | Decides & acts | Autocomplete suggestions |
| Automation | Acts | Approves | Draft email, human sends |
| Full automation | Acts | Monitors | Spam filter |

### Autonomy Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Appropriate autonomy | AI authority matches risk | Critical |
| Escalation path | High-stakes ‚Üí human review | Critical |
| Intervention points | User can stop/modify at checkpoints | Major |
| Audit trail | Actions logged for review | Minor |

## Error Handling for AI

### AI Error Types

| Error Type | User Experience | System Response |
|------------|-----------------|-----------------|
| Wrong output | AI gives incorrect answer | Correction mechanism |
| No output | AI can't process request | Fallback + guidance |
| Harmful output | AI produces inappropriate content | Filter + report |
| Hallucination | AI invents false information | Source verification |
| Bias | AI shows unfair patterns | Monitoring + mitigation |

### Error Handling Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Graceful degradation | Fallback when AI fails | Critical |
| Clear error messaging | Explains AI limitation | Major |
| Recovery path | Alternative without AI | Major |
| Retry mechanism | Easy to try again | Minor |
| Error reporting | Option to flag problems | Minor |

### Error Message Patterns

```
Generic AI failure:
"I couldn't complete that request.
 Try rephrasing or [use manual option]."

Content generation failure:
"I wasn't able to generate content for this.
 Here are some templates you can customize."

Recognition failure:
"I couldn't identify that clearly.
 Try with better lighting or a different angle."

Rate limit:
"I'm handling many requests right now.
 Please try again in a moment."
```

### Fallback Strategies

| AI Feature | Fallback | Implementation |
|------------|----------|----------------|
| Smart search | Basic search | Keyword matching |
| Auto-complete | No suggestions | Empty state gracefully |
| Content generation | Templates | Pre-written options |
| Image recognition | Manual input | Form fields |
| Voice input | Text input | Keyboard alternative |

## Bias & Fairness

### Bias Detection Points

| Area | What to Check | Severity |
|------|---------------|----------|
| Training data | Representative of all users | Critical |
| Output distribution | Fair across demographics | Critical |
| Error rates | Equal accuracy for all groups | Critical |
| Default assumptions | Neutral starting point | Major |
| Language | Inclusive terminology | Major |

### Fairness Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Demographic parity | Similar outcomes across groups | Critical |
| Equal opportunity | Similar true positive rates | Critical |
| Calibration | Confidence matches accuracy | Major |
| Counterfactual fairness | Decision same if demographics changed | Critical |

### Bias Mitigation UI

| Pattern | Description | Example |
|---------|-------------|---------|
| Diverse examples | Show variety in samples | Image recognition demo |
| Bias warnings | Flag potential issues | "Results may vary by..." |
| Inclusive defaults | Neutral starting options | Non-gendered defaults |
| User-defined criteria | Let user set fairness rules | Filter preferences |

## Performance & Latency

### Perceived Performance for AI

| Response Time | User Expectation | UI Pattern |
|---------------|------------------|------------|
| <500ms | Instant | No indicator needed |
| 500ms - 2s | Quick | Subtle loading indicator |
| 2s - 10s | Processing | Progress + context |
| >10s | Background task | Queue + notification |

### AI Loading States

| Pattern | When to Use | Example |
|---------|-------------|---------|
| Skeleton | Known output structure | Generated text placeholder |
| Progress bar | Measurable steps | "Analyzing... 3 of 5" |
| Status updates | Long operations | "Searching...", "Compiling..." |
| Streaming | Incremental output | ChatGPT-style word streaming |

### Performance Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Appropriate feedback | Loading matches duration | Major |
| Cancelable | Long operations can abort | Minor |
| Partial results | Show progress when possible | Minor |
| Timeout handling | Clear message if too long | Major |
| Background option | Queue for async processing | Minor |

## Conversational AI Specific

### Conversation UX Checklist

| Check | Pass Criteria | Severity |
|-------|---------------|----------|
| Clear boundaries | What AI can/cannot discuss | Major |
| Context retention | Remembers conversation | Major |
| Clarification requests | Asks when unclear | Major |

### Conversation Patterns

| Pattern | Description | Example |
|---------|-------------|---------|
| Disambiguation | "Did you mean A or B?" | Multiple interpretations |
| Confirmation | "To confirm, you want to..." | Before actions |
| Follow-up | "Would you also like..." | Related suggestions |
| Hand-off | "Let me connect you with..." | To human support |

```

‚ùå Claiming emotions or experiences
```
